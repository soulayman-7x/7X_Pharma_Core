نموذج إدارة دُفعات الأدوية (Batches). يتحكم في تتبع المخزون على مستوى كل دفعة، بما يشمل رقم الدفعة، تاريخ الصلاحية، والكمية الحالية. مرتبط ارتباطاً مباشراً بجدول batches في قاعدة البيانات.

📝 الكود الكامل

app/models/Batch.php PHP 
class Batch extends Model {
    protected $table = 'batches';
    
    public function getBatchesByMedicineId($medicine_id) {
        $sql = "SELECT * FROM {$this->table} 
                WHERE medicine_id = ? 
                ORDER BY expiry_date ASC";
        $stmt = $this->query($sql, [$medicine_id]);
        return $stmt->fetchAll(PDO::FETCH_ASSOC);
    }

    public function addStock($medicine_id, $batch_number, $expiry_date, $quantity) {
        return $this->insert([
            'medicine_id'       => $medicine_id,
            'batch_number'      => $batch_number,
            'expiry_date'       => $expiry_date,
            'current_quantity'  => $quantity
        ]);
    }
}

🔍 شرح الدوال بالتفصيل

🔹 getBatchesByMedicineId($medicine_id)

تُرجع قائمة بجميع دفعات دواء معين من جدول batches، مرتبةً تصاعدياً حسب تاريخ الصلاحية (الأقرب انتهاءً أولاً).

🎯 الهدف: عند عرض تفاصيل دواء ما، نحتاج لمعرفة الدفعات المتاحة ومتى تنتهي كل منها لاختيار دفعة البيع الصحيحة (FEFO: First Expired, First Out).

  • ✅ تستخدم Prepared Statement مع ? لحماية من SQL Injection
  • ✅ ORDER BY expiry_date ASC يضمن عرض الدفعات الأقرب انتهاءً أولاً
  • ✅ fetchAll(PDO::FETCH_ASSOC) تُعيد مصفوفة ترابطية لكل الصفوف

⚠️ ترتيب FEFO وأهميته

ORDER BY expiry_date ASC يُطبّق مبدأ FEFO (First Expired, First Out): أي الدفعة التي تنتهي صلاحيتها أولاً يتم بيعها أولاً. هذا معيار صيدلاني حيوي يقلل من الهدر.

في نظام البيع، يجب دائماً اختيار الدفعة التي تأتي أولاً في هذه القائمة لضمان تطبيق FEFO تلقائياً.

🔹 addStock($medicine_id, $batch_number, $expiry_date, $quantity)

تُضيف سجلاً جديداً في جدول batches لتسجيل وصول دفعة جديدة من دواء معين.

$medicine_id int معرّف الدواء المرتبط بهذه الدفعة من جدول medicines
$batch_number string الرقم التسلسلي للدفعة كما هو مذكور على العبوة (مثال: BT-2024-001)
$expiry_date string تاريخ الصلاحية بصيغة YYYY-MM-DD
$quantity int الكمية الابتدائية للدفعة عند الإضافة

تستدعي $this->insert() الموروثة من Model والتي تُنفّذ INSERT وتُعيد ID السجل الجديد.
