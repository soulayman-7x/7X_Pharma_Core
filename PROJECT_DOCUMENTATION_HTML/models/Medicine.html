💡 شرح مكونات الـ SQL:

  • LEFT JOIN batches — يربط الأدوية بدفعاتها. LEFT JOIN يضمن إظهار الدواء حتى لو لم يكن له دفعات
  • COALESCE(SUM(...), 0) — يجمع كل كميات الدفعات. إذا لم توجد دفعات يعيد 0 بدلاً من NULL
  • deleted_at IS NULL — لا يُظهر الأدوية المحذوفة حذفاً ناعماً (Soft Delete)
  • LIKE ? — بحث جزئي: %paracetamol% يجد أي كلمة تحتوي "paracetamol"
  • LOWER(...) = LOWER(?) — مقارنة غير حساسة لحالة الأحرف
  • GROUP BY medicines.id — ضروري مع SUM() لتجميع كل دفعات الدواء الواحد

💡 Soft Delete vs Hard Delete:

  • Hard Delete: DELETE FROM medicines — يحذف السجل نهائياً ← خطر: لا يمكن استعادة الفواتير التاريخية
  • Soft Delete: يضع تاريخ في عمود deleted_at — السجل موجود لكن مخفي في الاستعلامات العادية
  • جميع استعلامات Medicine تُضيف WHERE deleted_at IS NULL لإخفائه
