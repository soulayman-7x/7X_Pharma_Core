index.php & .htaccess

نقطة الدخول الوحيدة للتطبيق. كل طلب HTTP يمر عبر هذين الملفين — .htaccess يعترض الطلبات ويُعيد توجيهها، وindex.php يُشغّل التطبيق.

ملف .htaccess — Apache Rewrite Rules

1Options -Indexes
2RewriteEngine On
3
4RewriteBase /7X_PHARMA_CORE/public/
5
6RewriteCond %{REQUEST_FILENAME} !-d
7RewriteCond %{REQUEST_FILENAME} !-f
8
9RewriteRule ^(.*)$ index.php?url=$1 [QSA,L]

🔍 شرح كل سطر:

  • Options -Indexes — يمنع عرض محتويات المجلدات عند الوصول لمجلد بدون index.html
  • RewriteEngine On — تفعيل محرك إعادة الكتابة في Apache
  • RewriteBase — يحدد المسار الأساسي للمشروع على السيرفر المحلي
  • RewriteCond !-d — الشرط: الطلب ليس مجلداً فعلياً موجوداً
  • RewriteCond !-f — الشرط: الطلب ليس ملفاً فعلياً موجوداً
  • RewriteRule ^(.*)$ — إذا تحقق الشرطان، أعِد توجيه كل شيء لـ index.php مع الرابط كمتغير url
  • [QSA,L] — QSA = دمج Query String الأصلي، L = آخر قاعدة (توقف البحث)

⚡ ماذا يعني هذا عملياً؟

عندما يكتب المستخدم /pos في المتصفح، يُحوّله .htaccess إلى index.php?url=pos تلقائياً — هذا ما يجعل الروابط نظيفة وجميلة.

ملف index.php — Bootstrap التطبيق

1<?php
2session_start();
3
5require_once '../app/config/Constants.php';
6require_once ROOT_DIR . '/app/config/Database.php';
7require_once ROOT_DIR . '/app/core/Model.php';
8require_once ROOT_DIR . '/app/core/Controller.php';
9require_once ROOT_DIR . '/app/core/Router.php';
10
11// Activate the router, which will read the link and guide us.
12$router = new Router();

📋 ترتيب تحميل الملفات وسبب الترتيب:

1
session_start() — أول شيء دائماً. يُشغّل الجلسة ليتمكن النظام من حفظ بيانات المستخدم وعناصر السلة.
2
Constants.php — يُعرّف ROOT_DIR الذي يُستخدم في جميع require_once التالية. يجب أن يكون أولاً بمسار نسبي لأنه لا يوجد ROOT_DIR بعد.
3
Database.php — تعريف كلاس الاتصال بالقاعدة. يُحمَّل قبل Model لأن Model يحتاجه.
4
Model.php — الكلاس الأب للـ Models. يُحمَّل قبل Controller لأن Controller يستدعي Models.
5
Controller.php — الكلاس الأب للـ Controllers. يُحمَّل قبل Router لأن Router يُنشئ Controller.
6
new Router() — يُشغّل الراوتر الذي يحلل الرابط وينفذ الـ Controller المناسب.

✅ نمط تصميم مهم: Single Entry Point

كل طلب يمر عبر index.php واحد فقط — هذا النمط يُسمى "Front Controller Pattern" ويُتيح:

  • التحكم المركزي في الجلسة والتحميل
  • تطبيق الحماية والـ middleware في مكان واحد
  • هيكل نظيف ومنظم للمشروع

📊 الملفات المُحمَّلة وتسلسلها:

الترتيبالملفالسبب
1session_start()تفعيل الجلسة قبل أي شيء
2Constants.phpتعريف ROOT_DIR بمسار نسبي مباشر
3Database.phpتعريف كلاس Database للاتصال
4Model.phpالكلاس الأب — يعتمد على Database
