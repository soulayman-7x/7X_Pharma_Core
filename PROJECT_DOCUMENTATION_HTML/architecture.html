نمط معمارية MVC المخصص

يعتمد نظام 7X Pharma Core على معمارية MVC (Model-View-Controller) مطورة خصيصاً باستخدام لغة PHP (OOP). الهدف من هذا النمط هو فصل منطق العمل (Business Logic) عن واجهة المستخدم (UI)، مما يضمن:

  • قابلية الصيانة (Maintainability): سهولة تحديد وتعديل الأكواد دون التأثير على أجزاء أخرى.
  • الأمان (Security): حماية قواعد البيانات بتغليف الاستعلامات داخل Models بعيداً عن المتصفح.
  • تنظيم الكود (Code Organization): كل ملف له دور محدد وواضح.

دورة حياة الطلب (Request Lifecycle)

عندما يقوم المستخدم بطلب رابط معين، يمر الطلب عبر المراحل التالية بشكل تسلسلي:

1. Browser
HTTP Request
2. Router
توجيه الطلب
3. Controller
معالجة المنطق
4. Model
التواصل مع DB
5. View
إرسال HTML

يبدأ كل شيء من ملف public/index.php، والذي يقوم بتهيئة بيئة العمل، تشغيل الـ Sessions، واستدعاء ملفات الـ Core (مثل Database, Controller, Model) بالإضافة إلى استدعاء الكلاس Router.

آلية عمل الـ Router (الموجه)

الـ Router هو القلب النابض للنظام الذي يستقبل الرابط من المتصفح (عبر ملف .htaccess) ويقوم بتحليله وتوجيهه.

تدفق الـ Router:

  • استقبال الرابط: يقوم بتحليل $_GET['url'] وتقسيمه إلى مصفوفة (Array) باستخدام explode('/', $url).
  • تحديد المتحكم (Controller): العنصر الأول في المصفوفة يمثل اسم المتحكم (مثلاً: pos يصبح POSController). إذا لم يتم توفير متحكم، يتم استخدام POSController كقيمة افتراضية.
  • تحديد الدالة (Method): العنصر الثاني يمثل الدالة المراد تنفيذها داخل المتحكم. القيمة الافتراضية هي index.
  • المتغيرات (Parameters): ما تبقى من المصفوفة يتم تمريره كمتغيرات للدالة المحددة.
  • التنفيذ: يتم استدعاء الدالة تمرير المتغيرات باستخدام call_user_func_array().

تحليل الأمان (Security Analysis)

يتم استخدام الدالة filter_var($url, FILTER_SANITIZE_URL) داخل الـ Router لتنظيف الرابط من أي أكواد ضارة (XSS) قبل تمريره إلى النظام، كما يتم التأكد من وجود ملف المتحكم قبل استدعائه file_exists() لمنع ثغرات File Inclusion.

إدارة قواعد البيانات (Database & Models)

يتم الاتصال بقاعدة البيانات حصراً عبر كلاس Database الموجود في app/config/ والذي يستخدم تقنية PDO (PHP Data Objects).

  • الحماية من SQL Injection: جميع الـ Models تعتمد على الـ Prepared Statements (أي prepare() ثم execute()) لحماية النظام بشكل كامل من حقن SQL.
  • كلاس الـ Model الأساسي: يوفر كلاس Model.php في الـ core اتصالاً جاهزاً بقاعدة البيانات لترثه جميع النماذج الأخرى مثل (Medicine, Sale, Batch).
  • استقلالية النماذج: كل Model يمثل جدولاً في قاعدة البيانات ويتعامل مع الاستعلامات الخاصة به فقط، مما يعزز مبدأ Single Responsibility.

إدارة الجلسات والمصادقة (Session & Auth)

بما أن النظام يتعامل مع بيانات حساسة للصيدلية، تم بناء نظام مصادقة صلب يعتمد على الـ Sessions.

  • يبدأ الـ Session في ملف index.php قبل أي إرسال للبيانات للمتصفح.
  • يقوم AuthController بالتحقق من بيانات الدخول الواردة ومقارنة كلمة المرور المشفرة (باستخدام password_verify()) مع قاعدة البيانات.
  • عند نجاح تسجيل الدخول، يتم تخزين user_id و role و username داخل الـ $_SESSION.
  • Middleware وهمي: في كل Controller، يتم التحقق من وجود الـ Session (مثلاً عبر دالة checkAuth()) لمنع الوصول غير المصرح به للصفحات المحمية.

ملاحظات هندسية

التحسين المستقبلي: يمكن فصل منطق التحقق من تسجيل الدخول (Authentication Check) إلى كلاس Middleware مخصص يتم استدعاؤه في الـ Router لتقليل التكرار في الـ Controllers وجعل الكود أكثر نظافة (DRY Principle).
