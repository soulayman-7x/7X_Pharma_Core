Model.php

النموذج الأساسي (Base Model) للاتصال بقاعدة البيانات وتنفيذ عمليات CRUD

الرئيسية / Core / Model.php

وصف الملف

ملف Model.php يعتبر القناة الآمنة والموحدة لجميع كلاسات النماذج الأخرى في النظام للتواصل مع قاعدة البيانات. يوفر دوال عامة لعمليات (Create, Read, Update, Delete) مجهزة للعمل مع أي جدول في النظام بناءً على متغير $table الخاص بكل Model وارث له.

المتغيرات والدوال المحمية

protected $db

يحتفظ بكائن الاتصال بـ PDO للاستخدام داخل الكلاس والوراثة.

protected $table

اسم الجدول الذي سيتعامل معه النموج (يتم تحديده داخل الكلاسات الفرعية، مثلاً في Medicine يحدد كـ 'medicines').

public __construct()

يستدعي كلاس Database.php ويأخذ الاتصال عبر دالة getConnection() ويخزنه في $this->db.

protected query($sql, $params = [])

دالة مركزية مخصصة لتجهيز وتشغيل الاستعلامات بآلية Prepared Statements لتوفير أقصى درجات الحماية من هجمات SQL Injection.

دوال CRUD العامة (Methods)

public getAll()

الوظيفة: تجلب جميع السجلات من الجدول المرتبط بالموديل.

الاستعلام: SELECT * FROM {$this->table} ويرجع مصفوفة من السجلات باستخدام PDO::FETCH_ASSOC.

public getById($id)

الوظيفة: تجلب سجلاً واحداً بناءً على معرّف ID.

الاستعلام: SELECT * FROM {$this->table} where id = ?

public insert($data)

الوظيفة: دالة ديناميكية ذكية لإدخال صف جديد في الجدول بغض النظر عن عدد الأعمدة.

طريقة العمل خطوة بخطوة:

  • تقوم باستخراج المفاتيح array_keys($data) وتحويلها لسلسلة تمثل أسماء الأعمدة.
  • تصنع علامات استفهام ? (Placeholders) تتطابق مع عدد عناصر البيانات.
  • تكوّن الاستعلام الديناميكي INSERT INTO table (col1, col2) VALUES (?, ?).
  • تنفذ الاستعلام وتُرجع الـ lastInsertId() الخاص بالصف المُدرج.

public update($id, $data)

الوظيفة: دالة لتحديث سجل بناءً على المعرف، بنفس الديناميكية المستخدمة في insert.

طريقة العمل: تبني جملة col1 = ?, col2 = ? بشكل تلقائي عبر المرور على $data في حلقة تكرار foreach. تستخدم rtrim() لحذف الفاصلة الزائدة المتبقية في نهاية السلسلة. وتُرجع عدد الصفوف المتأثرة بعملية التحديث.

public delete($id)

الوظيفة: حذف سجل واحد بناءً على المعرف وترجع عدد الصفوف المحذوفة rowCount().

تحليل الأمان

الاعتماد المطلق على $this->db->prepare() في جميع الدوال يغلق تماماً أي احتمال لاختراق قواعد البيانات عبر إدخالات مستخدم خبيثة (SQLi)، حيث أن PDO يعزل البيانات عن أمر SQL ولا يفسرها كأكواد تنفيذية أبداً.
