💡 لماذا require_once داخل __construct وليس في الأعلى؟

لأن هذا الملف يُحمَّل في بداية التطبيق قبل تعريف ROOT_DIR أحياناً. داخل الـ constructor، ROOT_DIR يكون جاهزاً دائماً. كما أن require_once تضمن عدم تحميل الملف مرتين.

دالة query() — نواة الاستعلامات

13protected function query($sql, $params = []) {
14 $stmt = $this->db->prepare($sql);
15 $stmt->execute($params);
16 return $stmt;
17}

✅ Prepared Statements — الحماية من SQL Injection:

  • prepare(): يُرسل هيكل الـ SQL لقاعدة البيانات بدون البيانات
  • execute([$params]): يُرسل البيانات بشكل منفصل — لا يمكن أن تُفسَّر كـ SQL
  • علامة ? في الـ SQL هي placeholder يُستبدل بالقيمة بأمان

دالة insert() — الأكثر تعقيداً في الكلاس

46public function insert($data) {
48 $columns = implode(', ', array_keys($data));
51 $placeholders = implode(', ', array_fill(0, count($data), '?'));
52 $sql = "INSERT INTO {$this->table} ({$columns}) VALUES ({$placeholders})";
54 $this->query($sql, array_values($data));
57 return $this->db->lastInsertId();
58}

🔍 كيف تُبنى الـ SQL ديناميكياً؟

المدخل: ['name' => 'Paracetamol', 'price' => 15.5, 'category' => 'pain']

  • array_keys($data) → ['name', 'price', 'category']
  • implode(', ', ...) → "name, price, category"
  • array_fill(0, 3, '?') → ['?', '?', '?']
  • implode(', ', ...) → "?, ?, ?"
  • النتيجة: INSERT INTO medicines (name, price, category) VALUES (?, ?, ?)
  • array_values($data) → [15.5, 'Paracetamol', 'pain'] كمعاملات

دالة update() — بناء SET ديناميكياً

61public function update($id, $data) {
62 $setClause = "";
64 foreach (array_keys($data) as $key) {
65 $setClause .= "{$key} = ?, ";
66 }
67 $setClause = rtrim($setClause, ', ');
69 $sql = "UPDATE {$this->table} SET {$setClause} WHERE id = ?";
71 $values = array_values($data);
72 $values[] = $id; // id يُضاف في النهاية لـ WHERE
74 $stmt = $this->query($sql, $values);
75 return $stmt->rowCount();
76}

💡 تفصيل المثال:

update(5, ['name' => 'X', 'price' => 10])

  • setClause بعد loop: "name = ?, price = ?, " (فاصلة زائدة)
  • rtrim(..., ', '): يُزيل الفاصلة الزائدة → "name = ?, price = ?"
  • SQL: UPDATE medicines SET name = ?, price = ? WHERE id = ?
  • values: ['X', 10, 5] — الـ id يُضاف في النهاية ليطابق WHERE

⚡ $values[] = $id — لماذا يُضاف في النهاية؟

لأن الـ ? في SQL تُعوَّض بالترتيب. WHERE id = ? هو آخر ? في الاستعلام، لذا يجب أن يكون $id آخر عنصر في مصفوفة القيم.
