🔍 شرح الأكواد المعقدة:

⚡ السطر 14: ucfirst + file_exists للحماية

ucfirst($url[0]) يُحوّل "pos" إلى "Pos" ثم يُضاف "Controller" → "PosController"، ثم يتحقق من وجود الملف فعلياً قبل تحميله لمنع أي خطأ أو اختراق.

⚡ السطر 16: unset($url[0]) — تقليص المصفوفة

بعد تحديد الـ Controller، يُزيل العنصر [0] من المصفوفة حتى يبقى فقط الـ method والـ params في المصفوفة. هذا يُبسّط المعالجة اللاحقة.

⚡ السطر 37: call_user_func_array — الاستدعاء الديناميكي

هذه الدالة القوية تستدعي دالة ديناميكياً بناءً على اسمها كنص. المعادل المباشر هو: $controller->method($param1, $param2) لكن باستخدام متغيرات.

📖 مثال تطبيقي كامل:

الرابط: /inventory/edit/5

  • parseUrl() → ['inventory', 'edit', '5']
  • $url[0] = 'inventory' → Controller = 'InventoryController'
  • unset($url[0]) → ['edit', '5']
  • $url[1] = 'edit' → method = 'edit'
  • unset($url[1]) → ['5']
  • params = ['5']
  • call_user_func_array → InventoryController::edit('5')

🔒 ثلاث خطوات للأمان والتحليل:

  • rtrim($url, '/') — يُزيل الشرطة المائلة من نهاية الرابط: "pos/" → "pos"
  • FILTER_SANITIZE_URL — يُنظّف الرابط من الأحرف غير المسموح بها (حماية XSS)
  • explode('/', $url) — يُقسّم الرابط بالشرطة: "inventory/edit/5" → ['inventory','edit','5']
