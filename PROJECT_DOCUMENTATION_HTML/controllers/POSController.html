📋 الخطوات السبع داخل الـ Transaction:

1
INSERT sales — إنشاء رأس الفاتورة (user_id, client_id, total_amount, payment_method) وجلب sale_id الجديد
2
جلب بيانات الدواء — getById() و getOldestBatchId() للحصول على بيانات الـ snapshot والدفعة الأقدم
3
إنشاء JSON Snapshot — حفظ اسم الدواء وسعره وبياناته وقت البيع. يضمن بقاء الفاتورة صحيحة حتى لو تغيّرت بيانات الدواء لاحقاً
4
INSERT sale_items — إدراج كل بند من الفاتورة (sale_id, batch_id, quantity, unit_price, snapshot_data)
5
deductStock() — خصم المخزون بخوارزمية FIFO. إذا فشل → Exception تُوقف كل شيء
6
INSERT inventory_movements — تسجيل حركة المخزون لتتبع التاريخ (من أي دفعة، كم كمية، من قبل من)
7
تحديث الدين (اختياري) — إذا كانت طريقة الدفع "credit" → UPDATE clients SET credit_balance = credit_balance + total_amount

✅ لماذا beginTransaction/commit/rollBack؟

  • Transaction = إما كل العمليات تنجح معاً أو تفشل معاً
  • إذا فشل خصم المخزون في منتصف الفواتير → rollBack() يُلغي كل ما تم
  • يمنع حالات التناقض: فاتورة موجودة بدون خصم مخزون مثلاً

💡 اتصال PDO واحد مشترك — لماذا؟

$db = $medicineModel->getDb() — جميع العمليات تستخدم نفس اتصال PDO. هذا ضروري لأن Transaction تعمل على اتصال واحد فقط. اتصالات متعددة = Transactions منفصلة = لا حماية.

⚡ Snapshot JSON — الحفاظ على سجل الأسعار التاريخية:

json_encode(['name', 'price', 'barcode', 'dci']) — لو تغيّر سعر الدواء بعد الشراء، الفاتورة القديمة تبقى بالسعر القديم لأنه محفوظ في snapshot_data.
